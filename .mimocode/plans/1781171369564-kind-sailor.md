# aiTui 优化计划

## 目标
在保持功能稳定简洁的前提下，提升用户体验和代码质量。

---

## 优化项

### 1. 流式输出 (Streaming) ⭐ 最大 UX 提升

**现状**: `callLLM()` 等待完整响应后一次性输出，用户长时间看不到反馈

**改动**: `agent/agent.ts`
- `callLLM()` 改为支持 `stream: true` 的 fetch
- 解析 SSE (Server-Sent Events) 逐块输出文本
- 工具调用仍需等完整响应（function calling 无法流式）

**新增文件**: `utils/stream.ts`
- SSE 解析器（处理 `data: {...}\n\n` 格式）
- 保持零依赖

**改动点**:
```
agent/agent.ts:59-93  callLLM() 方法
agent/agent.ts:164-171  最终回答输出处
新增 utils/stream.ts  SSE 解析器
```

---

### 2. 终端颜色输出

**现状**: 纯文本，无视觉层次

**改动**:
- `cli/index.ts`: 帮助信息添加颜色（选项高亮、版本号）
- `agent/agent.ts`: 工具名、错误信息、成功提示加颜色
- `guard/safety.ts`: BLOCKED/DANGEROUS 消息用红/黄色

**新增文件**: `utils/color.ts`
- 简单的 ANSI 颜色函数（红/绿/黄/蓝/灰）
- 零依赖，直接写 ANSI escape code

---

### 3. 文档修正

**AGENTS.md** 过时内容:
- L27: 默认模型写的是 `DeepSeek-V3.1`，实际代码是 `glm-5`
- L117-123: 配置示例用的环境变量名是 `AI_API_URL`，实际是 `LLM_API_URL`
- L129-133: 添加工具步骤描述旧的双重注册模式，实际用 `tools/registry.ts`

**改动**: 直接修正 AGENTS.md 中的过时内容

---

### 4. 安全守卫补充

**现状**: 7 个 BLOCKED + 9 个 DANGEROUS 模式

**补充**:
- BLOCKED: `chmod 777 /`（危险权限设置）
- DANGEROUS: `systemctl restart`（误操作可影响服务）
- DANGEROUS: `crontab -r`（清空定时任务）

## 验证方式

1. **流式输出**: 运行 `ai "hello"`，观察文本是否逐字显示
2. **颜色输出**: 运行 `ai help`，确认选项有颜色高亮
3. **安全守卫**: 测试 `ai "chmod 777 /"` 应被 BLOCKED
4. **编译**: `bun run build:mac-arm` 生成新二进制
